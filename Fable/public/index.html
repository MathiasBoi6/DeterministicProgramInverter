The code to parse and lex which is otherwise used in the project does not work with Fable. Therefore the lexing on this webhost is suboptimal. Nonterminals of grammar need to surronded by double quotes, like this "inc". The terminal (nonterminal string), however, should not have the double quotes. The brackets [ and ] hold significant meaning for parsing, so nonterminals cannot have them. Do not use semicolon ; in grammar, just exclude it, signs such as '(' has no meaning in the grammar, but it does hold meaning for lexing the stack.
